User Guide Content

Welcome to the 3E Academic Program Assessment. This user guide will help you get started.

For quick reference:

The 3E SaaS platform can be accessed at: https://saas-stage.3enrollment.com

Technical support can be requested through the platform or by emailing software.support@3enrollment.com.

Getting Started

  1. To access the 3E SaaS platform, type in or paste https://saas-stage.3enrollment.com into your browser (Chrome or Edge are preferred browsers). Bookmark or favorite this URL.
  2. Enter your email and password credentials that were sent to you by 3E. If you do not have your credentials, you may email software.support@3enrollment.com and 3E will resend your credentials.
  3. Once logged in, you can manage your log-in credentials and create and manage additional users on your campus

Managing Licenses and Credentials

  1. From the home screen, locate the gear icon
  2. in the top right navigation menu.
  3. Click on the gear to select managing your profile or the profiles of additional users.
  4. Selecting Profile will allow you to update your password by entering and confirming your new password in the prompts.
  5. Selecting Users will allow you to create new and manage existing users:
  6. To create new users, click the Add New User button and follow the prompt, filling in username, email, and temporary password.
  7. There are two types of user roles: Client Admin and Client User. Checking the box titled Admin will create that user as a Client Admin. Leaving that bock unchecked will create that user as a Client User.
  8. Client Admins can create and manage users in addition to loading data and creating and managing run rate models and scenarios. Client Users can only load data and create and manage run rate models and scenarios.
  9. Editing a Client User will allow the Client Admin to change the user password and role.

SaaS Modules Overview

  1. This user guide will provide additional details on how to use these modules in subsequent sections of this user guide. First, let’s get familiar with the platform layout and navigation. From your home screen you can navigate to five different modules in the platform, as indicated with the five yellow and clickable circles:
  2. Executive Summary – reporting dashboard that shows the results of the run rate as well as supporting data that gives context to the run rate results.
  3. View 3E Enrollment Run Rate – in this module the user can see the population, college going, and mobility projections as well as the projecting conversion rates that lead to the run rate projections.
    • By clicking on the Detailed Run Rate button, the user can review the run rate data by state or by race/ethnicity to understand trends that shape their run rates.
    • Clicking on Run Rate by State provides a table that shows the population and market details as well as the anticipated conversion rates and funnel metrics based on your recent performance and anticipated market conditions by each state.
    • Clicking on Scenario Planning navigates the user to the Scenario Simulator (see Scenario Simulator section below).
  4. Create and Manage Run Rate Models – in this module the user can create new run rate models, delete unwanted run rate models, or favorite a model to view in the Executive Summary and Scenario Simulator. (See Creating Run Rate Models below).
  5. User Guide – the user how to section and includes a copy of this user guide.
  6. Scenario Simulator – allows the user to adjust anticipated population and conversion data to see the impact on future enrollment and net tuition revenue.
  7. Top Navigation Menu - allows the user to access/return to the home screen, manage their profile, refresh the page, seek help, and provide feedback or ideas for future releases.

Loading Data

  1. The 3E Run Rate uses data from the three most recently enrolled classes. For this reason, data needs to be loaded only once annually – typically after the fall census date. Here are the steps needed to load data directly into the platform:
  2. Refer to the 3E Run Rate Data Layout and the 3E Run Rate Data Sample File to make certain that you have the correct data format.
  3. From the Home Screen, click on the Load/Manage My Data link next to your name in the top navigation bar.
  4. Click the Choose File button.
  5. Select your file and click the blue Upload button.
  6. Once uploaded, the teal status bar will indicate how many records were loaded.
  7. Click on the Overall link to see the funnel counts of your historic data.
  8. If the counts are correct, approve your data by clicking the approve button on the right-hand side, directly under the gold header section.
  9. If the counts are incorrect, you may correct your data file and initiate the upload process with your corrected data.

Creating a Run Rate Model

  1. Once data is loaded into the platform, users may build run rate models.
  2. From Home Screen, click on the yellow menu circle titled Create and Manage Run rate Models to bring up the Run Rate Model menu.
  3. To create a new run rate model, click the Create New Model button on the right, underneath the yellow banner.
  4. Run rates are created using the client’s institutional historical data, 3E’s population and market variable data, and inputs from the user. The teal boxes on the Create New Model page are fields for the user to input:
    • Input year – while the run rate model will project seven future years, entering the input year allows the user to see that specific year in more detailed views in the Detailed Run Rate screen.
    • Input Weights on Conversion Rates – the run rate models use recent historic conversion as a component of the projection calculations. These fields allow the user to weight each of the three historic years to account for years that are anomalies. For example, a user may have deployed a strategy in 2022 that they will continue to use and may wish to increase the weight given to the 2022 conversion. Note – total weights across all three years must equal 1.0.
    • Application Goals – enter application goals here so that the run rate models may provide a comparison against future projections. Note, leaving these fields blank will still allow the run rate model to calculate.
    • Enrollment Goals - enter enrolled student goals here so that the run rate models may provide a comparison against future projections. Note, leaving these fields blank will still allow the run rate model to calculate.
    • Net Tuition Revenue - enter total freshmen net revenue goals here so that the run rate models may provide a comparison against future projections. Note, leaving these fields blank will still allow the run rate model to calculate.
  5. Click Save and name your run rate.
  6. Once saved you may see the new run rate model by clicking the Home Screen button and then clicking the yellow Create and Manage Run Rate Models.
  7. Find your new run rate and favorite it by clicking the star to the left of the run rate.
  8. To see you run rate output, click the Executive Summary button to the right of the listed run rate. Note: Favoriting a run rate will allow the user to see the run rate details in the Executive Summary as well as the View 3E Run Rate modules. The Scenario Simulator module also uses the favorited run rate.
